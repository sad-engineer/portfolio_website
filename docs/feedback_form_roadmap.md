### 5.0 Приоритетный пакет защиты (актуальный план)

Цель: резко снизить спам и авто-сбор данных через публичную форму.

Включаем:

1. **Cloudflare Turnstile**:
   - на фронтенде рендерить виджет и получать одноразовый токен;
   - в `POST /api/feedback` проверять токен через `https://challenges.cloudflare.com/turnstile/v0/siteverify`;
   - добавить env-переменные:
     - `FEEDBACK_TURNSTILE_SITE_KEY`
     - `FEEDBACK_TURNSTILE_SECRET_KEY`
   - если токен невалиден — возвращать `403` с безопасным текстом ошибки.

2. **Rate-limit на `POST /api/feedback` по IP**:
   - не больше 3 запросов за 10 минут;
   - минимальный интервал между отправками с одного IP — 2 минуты;
   - при превышении лимита возвращать `429 Too Many Requests`.

3. **Honeypot-поле в форме**:
   - скрытое поле (например, `fullname`), которое человек не заполняет;
   - если поле заполнено — считать запрос ботом, не отправлять в SMTP/Telegram.

Плюс: сильно режет спам и авто-сбор с формы.  
Минус: нужно чуть больше работы (frontend + backend + env + тесты).

### 5.1 Validation Policy для международных номеров

- Использовать библиотеку `phonenumbers` (Python port libphonenumber) как основной источник валидации.
- На входе разрешать международный формат (с `+`) и локальный формат (без `+`).
- Для номеров без `+` применять fallback-регион по умолчанию: `RU`.

Серверная обработка телефона:

1. Сохранить `phone_raw` как введено пользователем.
2. Выполнить нормализацию:
   - убрать пробелы, `(`, `)`, `-`, `.`;
   - разрешить `+` только в начале строки;
   - после очистки оставить только цифры и ведущий `+`.
3. Применить библиотечные проверки:
   - `is_possible_number` — реалистичность длины/структуры;
   - `is_valid_number` — валидность номера по плану нумерации страны.
4. При успехе сохранить `phone_e164` в формате `+<countrycode><nationalnumber>`.
5. При ошибке вернуть пользователю локализованное сообщение о невалидном номере.

Критерии "мусорного номера" (дополнительно к library validation):

- после нормализации меньше 8 или больше 15 цифр;
- все цифры одинаковые (`00000000`, `11111111`, ...);
- монотонные последовательности (`12345678`, `87654321`);
- короткий повторяющийся паттерн (`12121212`, `123123123`);
- значения из blacklist тестовых шаблонов (`1234567890`, `9999999999` и т.п.).

Примечание по UX:

- На фронтенде маску не фиксировать под одну страну.
- Разрешать свободный ввод международного номера и валидировать окончательно на сервере.
